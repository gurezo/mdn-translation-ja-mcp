# L10N ガイドライン

## 基本方針

Mozilla 日本語 L10N は、製品が日本語化されたと感じさせず、自然な日本語で使えるようにする。

## 言語差と意訳

- 英語と日本語の語順・主述の違いに注意し、逐語訳を避ける
- 英語直訳特有の言い回しを避ける
- 余分な主語は削除する
- 原文が不適切・不明瞭な場合は独自表現に変えてよい

### 翻訳例

| 原文 | 不適切 | 適切 |
| --- | --- | --- |
| Update Succeeded. | 更新成功 / 更新に成功しました。 | 更新を正常に完了しました。 |
| You can click on this icon to see… | あなたは…クリックすることができます。 | このアイコンをクリックすると…確認したり…許可したりできます。 |

## 文体と表現

- 敬体（です・ます調）で統一
- 必要以上の漢字は使用しない
- 固有名詞・専門用語・慣例を除き英語表記は使用しない
- カタカナ語も可能な限り自然な日本語に置き換える
- 「Web」は「ウェブ」（2016 年 8 月以降）
- 例外: Cookie、DOM などは英語表記のまま

## コンテクスト別表現

| UI 要素 | 原則 |
| --- | --- |
| ウィンドウタイトル | 体言止め |
| メニュー項目 | OS 慣用に従い、簡潔な体言 |
| カテゴリー名 | 体言止め。コロン付けや説明文への続けは避ける |
| ボタンラベル | 動作・目的を簡潔な体言で。可能なら語数を揃える |
| ツールチップ | 動作と目的語を含め、何をどう処理するか分かる表現 |
| チェックボックス | 末尾は動詞を原則。複文を除き句点なし |
| ラジオボタン | 前後文脈に注意。末尾品詞は問わない |
| プルダウンメニュー | 共通部は説明部に、各項目は簡潔に |
| 本文テキスト | 文脈判断。文になるところでは句点を使用 |

## 英数記号と全角半角

- `(` の前は半角空白、`)` の前は詰める、`)` の後は半角空白
- `:` の前は詰め、後は半角空白
- 例（現行）: `Firefox は Mozilla プロジェクト (mozilla.org) による次世代ウェブブラウザーです！`

## 類似表現とかな漢字

- 「すべて」はひらがな
- 「～することが～」はひらがな
- 「～することができます」→「～できます」と簡潔に
- 「～するとき」はひらがな、「～する場合」は漢字
- 「～のみ」（文語調）と「～だけ」（口語調）を使い分ける

## その他

- OS によって訳語が異なるものは訳し分ける
- 初心者向け箇所は説明的に
- 技術的に正確な表現を心がける
